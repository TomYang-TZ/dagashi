Pixel Art as a Third Visual Mode

Alongside ASCII (monochrome) and ASCII (color), add pixel art as a distinct output. Each pull randomly gets one of three styles. Pixel art renders as actual colored blocks in the webview — no text characters needed.

B

Pixel Art Replaces ASCII Entirely

Ditch the brightness→character mapping. Every pull is pixel art with palette-mapped colors rendered as colored blocks (or colored characters). Simpler pipeline, stronger visual identity.

Neon Pop palette
Ocean palette
C

Pixel Art + ASCII Hybrid

Render pixel art but use the user's typed characters as the "pixels" — each block is a colored character from their keystroke stats. Merges both concepts: you see the pixelated image AND your typing fingerprint.
